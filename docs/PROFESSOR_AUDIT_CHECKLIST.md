# Professor / Reviewer Audit Checklist

A reviewer can inspect this repository by checking:

1. Are the definitions finite and explicit?
2. Are the boundary maps dimension-compatible?
3. Do the chain conditions hold where claimed?
4. Are obstruction dimensions computed by standard finite linear algebra over \(\mathbb F_2\)?
5. Are examples reproducible from public JSON inputs?
6. Are receipts deterministic and replayable?
7. Are global open problems explicitly not claimed solved?
8. Does GitHub Actions replay the verifier?

Passing this checklist does not prove any global conjecture; it verifies the finite framework.
